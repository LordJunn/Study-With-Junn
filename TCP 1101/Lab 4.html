Removing the break statements in the switch cases will significantly impact the program's execution. 
Here's what happens:

    Without breaks: 
    When a case is matched (e.g., grade is A), the corresponding statement block (cout statement) executes. 
    However, due to the lack of break, it doesn't stop there.
    
    Fall-through: 
    Instead, it automatically falls through to the next case, even if it doesn't match the grade value. 
    This means subsequent statements will also be executed one after the other, until the end of the switch statement or a break is encountered.
    
    Unexpected output: 
    Therefore, the program might output messages for multiple grades instead of just the matching one. 
    For example, if grade is A, it will print "an A - excellent work!" followed by all subsequent messages: 
    "you got a B - good job", "earning a C is satisfactory", and so on.

Overall, removing the break statements transforms the switch into a cascade. 
It becomes more like a series of if-else statements chained together, 
where each case adds its message to the output if the previous cases are also true. 
This can lead to confusing and potentially misleading results for the user as they might not receive a clear and concise evaluation of their grade.

Remember, adding a break after each case statement restores the expected behavior where only the matching message is printed.

I hope this clarifies the effect of removing the break statements and helps you understand the importance of their role in switch cases.


If you remove the break statements from each of the cases in your switch statement, the program will exhibit what is known as “fallthrough” behavior. 
This means that once a matching case is found, the program will execute not only the code for that case, 
but also the code for all subsequent cases, until it either reaches a break statement or the end of the switch block.

This is likely not the intended behavior. 
The break statement is used in each case to prevent this fallthrough behavior and ensure that only the code for the matching case is executed.

  double discount = 0.0;

    discount = 0.2;

    discount = 0.25;

  // Calculate price with discount
  double pricePerShirt = BASE_PRICE * (1.0 - discount);

  // Display output with appropriate pricing
  cout << fixed << setprecision(2);
